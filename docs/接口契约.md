# 多代测序组装基准接口契约

本文档是扩展 benchmark 的统一接口。所有脚本和成员产物必须按这里的命名、路径和 CSV schema 对接，避免人工改名。

## Run ID

统一格式：

```text
{tech}_{cov}x_{assembler}_{ref}_s{seed}
```

示例：

```text
ont_r10_30x_flye_orig_s13
hifi_30x_hifiasm_orig_s13
illumina_10x_spades_rep_v1_s17
```

字段含义：

| 字段 | 示例 | 含义 |
|---|---|---|
| `tech` | `illumina`, `ont_r9`, `ont_r10`, `hifi`, `hybrid` | 测序技术或混合策略 |
| `cov` | `5`, `10`, `20`, `30`, `50` | 标称覆盖度 |
| `assembler` | `spades`, `flye`, `raven`, `miniasm`, `hifiasm`, `unicycler` | 组装工具 |
| `ref` | `orig`, `rep_v1`, `rep_v2`, `plasmid` | 参考基因组场景 |
| `seed` | `13`, `17`, `19` | 随机种子 |

## 输入路径

参考基因组：

```text
data/reference/ecoli.fasta
data/reference/ecoli_rep_v1.fasta
data/reference/ecoli_rep_v2.fasta
data/reference/ecoli_plasmid.fasta
```

Reads：

```text
data/reads/{tech}_{cov}x.fastq
data/reads/illumina_{cov}x_1.fastq
data/reads/illumina_{cov}x_2.fastq
```

## 组装输出路径

每个 run-id 一个目录：

```text
assemblies/{run_id}/assembly.fasta
assemblies/{run_id}/assembly_graph.gfa
```

`assembly_graph.gfa` 只在组装器能够导出图时存在。

## 主指标表

主表路径：

```text
results/tables/assembly_metrics.csv
```

主表列顺序：

```text
run_id,tech,generation,coverage,assembler,reference,seed,n50,contigs,largest_contig,total_length,genome_fraction_pct,mismatches_per_100_kbp,indels_per_100_kbp,misassemblies,busco_complete_pct,merqury_qv,ram_mb,wall_sec
```

说明：

| 列 | 含义 |
|---|---|
| `run_id` | 唯一实验 ID |
| `tech` | 测序技术 |
| `generation` | 技术代次，例如 `ont_r10` |
| `coverage` | 标称覆盖度 |
| `assembler` | 组装工具 |
| `reference` | 参考场景 |
| `seed` | 随机种子 |
| `n50` | QUAST N50，bp |
| `contigs` | QUAST 统计的 contig 数 |
| `largest_contig` | 最大 contig 长度，bp |
| `total_length` | 总组装长度，bp |
| `genome_fraction_pct` | 参考基因组覆盖百分比 |
| `mismatches_per_100_kbp` | 每 100 kbp mismatch 数 |
| `indels_per_100_kbp` | 每 100 kbp indel 数 |
| `misassemblies` | QUAST misassembly 数 |
| `busco_complete_pct` | BUSCO 完整基因百分比 |
| `merqury_qv` | Merqury QV，无参准确度指标 |
| `ram_mb` | 峰值内存，MB |
| `wall_sec` | 墙钟时间，秒 |

## 配置文件

矩阵唯一事实源：

```text
config/matrix.yaml
```

随机种子：

```text
config/seeds.yaml
```

脚本必须优先从这两个文件读取实验轴，不应在代码里散落硬编码矩阵。
