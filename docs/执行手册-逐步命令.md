# 执行手册 · 逐步命令（你照着跑）

> 配套：`docs/report.md`（最终报告与结果）、`docs/execution_status.md`（执行状态与证据）、`paper/main.tex`（ECCV 论文）。
> 本文档是**可直接复制粘贴的命令清单**。脚本已写好且单元测试全过，但**实验工具没装、矩阵一组没跑**——本手册带你从第 0 步把真实实验跑出来。

## ⚠️ 三条必读前提

1. **这些实验脚本从没对真实工具跑过**，第一次真跑大概率会冒出小 bug。**每个 Phase 先做"冒烟"（只跑 1 个配置），跑通了再放大到全矩阵。** 不要一上来就跑全量。
2. 命令里的 Python 一律用环境内解释器：`.conda/bench/bin/python`（CPU）/ `.conda/gpu/bin/python`（GPU）。**都从项目根目录运行**（`scripts` 才能被 import）。
3. 现在的 `coverage_genome_fraction_curve.png` / `pareto_quality_cost.png` 是**只有 1 个覆盖度的假图**，跑完真矩阵后会在 Phase 5 被覆盖。**在那之前别拿去答辩。**

约定：每步末尾的 **✅检查点** 是"过了才往下走"的硬条件。

> 逐个把 `flye / raven / miniasm / spades / unicycler` 各冒烟一次，确认 6 个分支都能产出 `assembly.fasta`（raven/miniasm/hifiasm 是 GFA→fasta，特别要看这步对不对）。

- [ ] **2.2 全量 L1 主网格（orig，约 50 组）**

```bash
.conda/bench/bin/python - <<'PY' 2>&1 | tee logs/assemble_L1.log
import yaml
from scripts.benchmark_common import expand_matrix
from scripts.run_assemblers import run_one
cfg = yaml.safe_load(open("config/matrix.yaml"))
seed = yaml.safe_load(open("config/seeds.yaml"))["default_seed"]
for r in expand_matrix(cfg, seed):
    if r["reference"] != "orig":      # 只跑 L1；L3 在 Phase 4.4
        continue
    if r["tech"] == "hybrid":         # hybrid 短+长前缀不同，见 2.3 单独处理
        continue
    try:
        rid = run_one(r["tech"], r["coverage"], r["assembler"],
                      r["reference"], r["seed"], threads=8)
        print("OK", rid)
    except Exception as e:
        print("FAIL", r, repr(e))
PY
echo "组装目录数：$(ls assemblies/ | wc -l)"
```

- [ ] **3.3 全量评估（遍历所有 assembly 目录）**

```bash
.conda/bench/bin/python - <<'PY' 2>&1 | tee logs/evaluate.log
import subprocess
from pathlib import Path
from scripts.benchmark_common import parse_run_id
for d in sorted(Path("assemblies").iterdir()):
    asm = d / "assembly.fasta"
    if not asm.is_file():
        continue
    try:
        rid = d.name; ctx = parse_run_id(rid)
    except ValueError:
        continue  # 跳过旧目录(hifi_30x 等非 run-id 命名)
    ref = {"orig":"ecoli","rep_v1":"ecoli_rep_v1","rep_v2":"ecoli_rep_v2",
           "plasmid":"ecoli_plasmid"}[ctx["reference"]]
    subprocess.run([".conda/bench/bin/python","scripts/evaluate_assembly.py",
        "--run-id",rid,"--assembly",str(asm),
        "--reference",f"data/reference/{ref}.fasta",
        "--out-dir",f"quast_reports/{rid}","--threads","8"], check=False)
PY
echo "主表行数：$(($(wc -l < results/tables/assembly_metrics.csv)-1))"
cut -d, -f4 results/tables/assembly_metrics.csv | sort -u   # 覆盖度应有 5/10/20/30/50
```

✅**检查点 3：** `assembly_metrics.csv` 行数 ≈ 组装数；`coverage` 列出现 **5 个不同值**（不再只有 30）；`busco_complete_pct` 列大量有值。**这是项目"不水"的硬证据。**

## Phase 4 — 机制 / GPU / 难点（线 B+D，~2–4 h）

- [ ] **4.1 GPU 抛光矩阵（创新点 5，线 D）**

```bash
# 冒烟：先对一个 ONT 组装跑 medaka，另开终端 nvidia-smi 确认吃显存
.conda/gpu/bin/python scripts/run_polish.py \
  --run-id ont_r10_30x_flye_orig_s13 \
  --assembly assemblies/ont_r10_30x_flye_orig_s13/assembly.fasta \
  --reads data/reads/ont_r10_30x.fastq --method medaka --rounds 1
# 全量：对所有 ont_* 的 flye 组装跑 {racon, medaka×1, medaka×2}
for d in assemblies/ont_*_flye_orig_s13; do
  rid=$(basename "$d")
  .conda/gpu/bin/python scripts/run_polish.py --run-id "$rid" \
    --assembly "$d/assembly.fasta" \
    --reads "data/reads/$(echo $rid | sed -E 's/_flye.*//').fastq" \
    --method medaka --rounds 2
done
cat results/tables/polish_metrics.csv | head
```

- [ ] **4.4 难点：重复 / 质粒（创新点 3/7，线 A+B，固定 30×）**

```bash
# 用变体参考模拟 30× reads（illumina/ont_r10/hifi 三种，看谁跨得过重复/拼得出质粒）
for ref in rep_v1 rep_v2 plasmid; do
  f=data/reference/ecoli_${ref}.fasta
  .conda/bench/bin/art_illumina -ss HS25 -i "$f" -p -l 150 -f 30 -m 350 -s 50 -rs 13 \
     -o data/reads/illumina_30x_${ref}
  .conda/bench/bin/badread simulate --reference "$f" --quantity 30x --seed 13 \
     --length 10000,5000 --identity 99,2,99.9 > data/reads/ont_r10_30x_${ref}.fastq
  .conda/bench/bin/badread simulate --reference "$f" --quantity 30x --seed 13 \
     --length 15000,3000 --identity 99.9,0.5,99.99 --error_model random --qscore_model random \
     > data/reads/hifi_30x_${ref}.fastq
done
# 组装（举例 ont_r10 + flye；illumina+spades、hifi+hifiasm 同理换 --assembler/--tech）
for ref in rep_v1 rep_v2 plasmid; do
  .conda/bench/bin/python scripts/run_assemblers.py --tech ont_r10 --coverage 30 \
    --assembler flye --reference $ref --seed 13 --threads 8 \
    --reads-prefix data/reads/ont_r10_30x_${ref}
done
# 评估时对应到各自的变体参考（见 Phase 3.3 的循环已按 reference 自动选参考）
```
✅ 得到"重复区/质粒"上三技术的对比；短读长在质粒/重复上的劣势可见。
