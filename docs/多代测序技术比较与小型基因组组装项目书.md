# 面向低算力场景的多代测序技术比较与小型基因组组装质量评估

**英文题目：** Comparative Evaluation of Multi-generation Sequencing Technologies for Small Genome Assembly in Low-computing-resource Scenarios

---

## 0. 项目一句话说明

本项目要做的事情很简单：

> 拿一条已知的小型基因组作为“标准答案”，在电脑上模拟不同测序技术产生的 DNA reads，然后用组装软件把 reads 拼回基因组，最后比较哪种测序技术在低算力条件下拼得更完整、更准确、更省资源。

通俗类比：

```text
原始基因组 = 一本完整的书
测序 reads = 被撕碎后读出来的小纸条
基因组组装 = 根据小纸条之间的重叠关系，把书重新拼回来
组装质量评估 = 看拼回来的书是不是完整、顺序对不对、错别字多不多
```

---

## 1. 项目定位

### 1.1 对应课程选题方向

本项目对应课程课设方向中的：

> 基因组学的探索和应用：多代基因测序技术的比较和应用评估。

本项目不是纯综述，而是一个完整的低算力实证实验：

```text
参考基因组
    ↓
模拟不同测序技术产生的 reads
    ↓
用不同组装工具拼接 reads
    ↓
用 QUAST 评价组装质量
    ↓
比较不同测序策略的优劣
```

### 1.2 为什么适合我们组

我们组目前没有生物信息学基础，只有 AI / 编程方向基础。这个项目适合我们，是因为它本质上是一个 **benchmark 项目**。

和 AI benchmark 很像：

```text
同一个标准答案
不同输入数据类型
不同算法流程
统一评价指标
最后比较性能
```

我们不需要自己做湿实验，不需要训练大模型，也不需要昂贵 GPU。只要能安装工具、跑命令、整理指标、画图和解释结果，就能完成。

## 2. 项目核心研究问题

本项目要回答的问题是：

> 在资源受限条件下，不同测序技术产生的 reads 特性如何影响小型基因组组装质量？

拆成 5 个小问题：

1. **Illumina 二代短读长 reads** 很准确，但是否会导致组装结果更碎？
2. **Nanopore 三代长读长 reads** 很长，但错误率高，是否会影响最终准确性？
3. **PacBio HiFi-like reads** 又长又准，是否能在低算力条件下得到更好的组装质量？
4. 测序深度从 **10× 到 30× 到 50×**，组装质量是否明显提升？
5. 如果只有普通电脑，应该优先选择哪种测序策略？

## 3. 零基础概念说明

本节给完全没有生信基础的同学看。先把这些词搞清楚，后面才不会乱。

### 3.1 Genome：基因组

基因组就是一个生物体完整的 DNA 序列。

例如：

```text
ATGCGTACCGTTAACCGG...
```

本项目建议使用：

```text
E. coli K-12 MG1655
```

也就是经典大肠杆菌参考基因组。它大约 4.6 Mb，比较小，普通电脑能跑。

---

### 3.2 FASTA

FASTA 是保存“标准答案基因组”的格式。

长这样：

```text
>NC_000913.3 Escherichia coli K-12 MG1655
AGCTTTTCATTCTGACTGCAACGGGCAATATGTCTCTGTGT...
```

第一行是名字，从 `>` 开始。后面是 DNA 序列。

---

### 3.3 Reads

reads 是测序仪读出来的一小段 DNA 序列。

例如原始 DNA 是：

```text
ATGCCGTAACCGTTAGCTA
```

测序仪可能读出这些 reads：

```text
ATGCCGTA
CCGTAACC
AACCGTTA
GTTAGCTA
```

这些小片段互相有重叠，后续组装算法就根据重叠关系把它们拼回去。

---

### 3.4 FASTQ

FASTQ 是保存 reads 的格式。

长这样：

```text
@read_001
ATGCCGTAACCG
+
IIIIHHHHGGGG
```

含义：

| 行 | 含义 |
|---|---|
| 第 1 行 | read 的编号 |
| 第 2 行 | 读到的 DNA 序列 |
| 第 3 行 | 分隔符 |
| 第 4 行 | 每个碱基的质量分数 |

质量分数越高，表示测序仪越确信这个碱基读对了。

---

### 3.5 Coverage：测序深度

coverage 可以理解为：

> 每个碱基平均被 reads 覆盖多少次。

例如：

```text
10× = 每个位置平均被 reads 覆盖 10 次
30× = 每个位置平均被 reads 覆盖 30 次
50× = 每个位置平均被 reads 覆盖 50 次
```

coverage 越高，数据越多，组装通常越可靠，但运行时间和内存消耗也会增加。

---

### 3.6 Assembly：基因组组装

基因组组装就是把 reads 拼回长序列。

常见思路：

| 类型 | 通俗解释 | 适合数据 |
|---|---|---|
| OLC | 找 reads 之间的重叠关系 | 长读长 |
| de Bruijn 图 | 把 reads 切成 k-mer，再找路径 | 短读长 |

本项目不需要自己写组装算法，只需要调用现成工具。

---

### 3.7 Contig

contig 是组装软件拼出来的一段连续 DNA。

例如组装结果是：

```text
contig_1: 2,000,000 bp
contig_2: 1,500,000 bp
contig_3: 1,100,000 bp
```

说明原始基因组被拼成了 3 段。

一般来说：

```text
contig 越少越好
contig 越长越好
```

---

### 3.8 N50

N50 是组装质量里最常见的指标之一。

通俗理解：

> N50 越大，说明拼出来的大块越多，组装越连续。

我们不需要手算 N50，QUAST 会自动算。

## 5. 实验设计

### 5.1 参考基因组

首选：

```text
E. coli K-12 MG1655
```

优点：

| 优点 | 原因 |
|---|---|
| 小 | 约 4.6 Mb，普通电脑能跑 |
| 经典 | 生物信息学教程常用 |
| 数据公开 | NCBI 可下载 |
| 不太简单 | 比病毒基因组更像真实组装问题 |

不要做人类基因组。人类基因组太大，不适合一周课程项目。

---

### 5.2 模拟三类 reads

| 测序类型 | 模拟对象 | 特点 | 项目中代表什么 |
|---|---|---|---|
| Illumina-like | 二代短读长 | 短、准 | 准确但容易碎 |
| Nanopore-like | 三代长读长 | 长、错多 | 连续但噪声高 |
| PacBio HiFi-like | 高准确长读长 | 长、准 | 理想长读长 |

---

### 5.3 组装工具

建议使用 3 个工具：

| 工具 | 用途 | 为什么用 |
|---|---|---|
| SPAdes | Illumina 短读长组装 | 经典细菌基因组组装工具 |
| Flye | Nanopore / HiFi 长读长组装 | 长读长组装常用工具 |
| Unicycler | 短读长 + 长读长混合组装 | 适合细菌 hybrid assembly |

---

### 5.4 评价工具

使用：

```text
QUAST
```

QUAST 会输出：

```text
N50
contig 数量
largest contig
total length
genome fraction
mismatches
indels
```

### 第 3 步：模拟 Illumina-like reads

```bash
art_illumina \
  -ss HS25 \
  -i data/reference/ecoli.fasta \
  -p \
  -l 150 \
  -f 30 \
  -m 350 \
  -s 50 \
  -o data/reads/illumina_30x_
```

参数解释：

| 参数 | 意思 |
|---|---|
| `-ss HS25` | 使用 Illumina HiSeq 2500 模型 |
| `-i` | 输入参考基因组 |
| `-p` | paired-end 双端 reads |
| `-l 150` | read 长度 150 bp |
| `-f 30` | coverage 30× |
| `-m 350` | 插入片段平均长度 |
| `-s 50` | 插入片段标准差 |

### 第 4 步：模拟 Nanopore-like reads

使用 Badread。

示例命令：

```bash
badread simulate \
  --reference data/reference/ecoli.fasta \
  --quantity 30x \
  --length 10000,5000 \
  > data/reads/nanopore_30x.fastq
```

### 第 5 步：模拟 HiFi-like reads

还是使用 Badread，但把错误率设置得更低。

示例命令：

```bash
badread simulate \
  --reference data/reference/ecoli.fasta \
  --quantity 30x \
  --length 15000,3000 \
  --identity 99.5,99.9,5 \
  > data/reads/hifi_30x.fastq
```

写 Python 脚本统计每个 FASTQ 的：

```text

```

```text

```

```bash
flye \
  --nano-raw data/reads/nanopore_30x.fastq \
  --out-dir assemblies/nanopore_30x \
  --threads 4 \
  --genome-size 4.6m
```

### 第 3 步：HiFi-like 长读长组装

使用 Flye：

```bash
flye \
  --pacbio-hifi data/reads/hifi_30x.fastq \
  --out-dir assemblies/hifi_30x \
  --threads 4 \
  --genome-size 4.6m
```

```bash

  --pacbio-corr data/reads/hifi_30x.fastq \
  --out-dir assemblies/hifi_30x \
  --threads 4 \
  --genome-size 4.6m
```

### 第 4 步：Hybrid 组装

使用 Unicycler：

```bash
unicycler \
  -1 data/reads/illumina_30x_1.fq \
  -2 data/reads/illumina_30x_2.fq \
  -l data/reads/nanopore_30x.fastq \
  -o assemblies/hybrid \
```

## 14. 每个人答辩时说什么

### A 的话术

> 我主要负责数据准备和 reads 模拟。我们首先选择 E. coli K-12 MG1655 作为小型参考基因组，然后分别模拟 Illumina-like、Nanopore-like 和 HiFi-like 三类 reads。Illumina-like reads 短但准确，Nanopore-like reads 长但错误率较高，HiFi-like reads 则模拟长且准确的数据。这样可以在同一个标准答案基因组上控制变量，比较不同测序技术对组装质量的影响。

### B 的话术

> 我主要负责组装流程。根据 reads 类型不同，我们采用不同组装工具：短读长 Illumina 数据使用 SPAdes，长读长 Nanopore-like 和 HiFi-like 数据使用 Flye，短读长和长读长混合数据使用 Unicycler。这样对应了短读长适合 de Bruijn 图思路，长读长适合 overlap-based 思路。

### C 的话术

> 我主要负责组装质量评估。我们使用 QUAST 对不同组装结果进行统一评价，指标包括 N50、contig 数量、largest contig、genome fraction、mismatch 和 indel。N50 越高说明组装越连续，contig 数越少说明结果越不碎，mismatch 和 indel 越少说明准确性越好。

### D 的话术

> 我主要负责论文、PPT 和结果解释。我们最终比较了不同测序技术在连续性、准确性和资源消耗之间的权衡。短读长准确但容易碎，长读长可以提高连续性但受错误率影响，HiFi-like reads 综合表现更平衡。项目最终给出了低算力条件下小型基因组组装的测序策略建议。

## 15. 风险与备选方案

### 风险 1：工具装不上

优先用 conda：

```bash
conda create -n genome_asm python=3.10 -y
conda activate genome_asm
conda install -c bioconda -c conda-forge spades flye quast unicycler badread art -y
```

如果 ART 装不上，用 wgsim 或自己写简单 Python 脚本模拟短 read。

### 风险 2：E. coli 跑太慢

备选换成更小的基因组：

```text
SARS-CoV-2，约 30 kb
```

但 SARS-CoV-2 太小，结果可能过于简单，只适合作为救命备选。

### 风险 3：Hybrid 跑不通

直接砍掉 hybrid。主实验保留：

```text
Illumina-like vs Nanopore-like vs HiFi-like
```

这三类已经足够支撑“多代测序技术比较”。

### 风险 4：结果不符合预期

不要慌。结果不完美也能解释。

例如：

```text
如果 Nanopore-like 的 N50 最高但错误也最高：
这正好说明长读长提升连续性，但错误率影响准确性。

如果 HiFi-like 各方面最好：
这说明高准确长读长在小型基因组组装中具有优势。

如果 Illumina-like 拼得很碎：
这符合短读长难以跨越重复区域的预期。
```

> 短读长准确但容易碎；长读长连续但错误多；HiFi-like reads 综合表现更平衡；hybrid 方案可能在连续性和准确性之间取得折中。我们用模拟 reads 和 QUAST 指标，在低算力条件下复现并验证了多代测序技术对小型基因组组装质量的影响。
