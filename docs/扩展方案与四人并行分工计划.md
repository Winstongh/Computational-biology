## 1. 为什么必须扩展（诊断）

现状结果在 30× 下**除 Illumina 外全部拼成 1 条完美 contig**（见 `results/tables/assembly_metrics.csv`），三个技术挤在天花板上，没有可比的梯度——这是"内容显水"的根因。

升级的四把钥匙：

1. **重新制造梯度** —— 加"低覆盖度轴（5×/10×）"和"重复区轴"，把饱和的问题重新拉开差距。
2. **从单点到多轴** —— 把"技术"和"工具""覆盖度""基因组复杂度"拆成正交的轴。
3. **从连续性到实用价值** —— 加无参评估 + 下游任务（基因恢复、变异检测），回答"高 N50 到底能不能用"。
4. **从跑命令到可复现 benchmark** —— 整条流水线一键重跑，benchmark 本身成为交付物。

## 2. 升级后的研究问题

1. 在 **5×–50× 覆盖度梯度**下，各技术达到"单 contig / 99% genome fraction"的**最小够用覆盖度**是多少？（饱和曲线 + 拐点）
2. 组装质量差异里，**多少来自"测序技术"、多少来自"组装工具"**？（技术 × 工具因子设计的方差归因）
3. "三代测序"内部，**ONT 自身从 R9(~90%) 到 R10(~99%) 的精度进步，能追平 HiFi 多少**？
4. **重复区域 / 质粒**这类难点，长读长凭什么赢？需要多长的 read 才能跨过去？（机制）
5. **GPU 神经网络抛光**能让廉价 ONT 数据在多大程度上、花多少算力代价追平 HiFi？
6. 把"质量"升维为"**能不能干下游活**"（基因恢复、变异检测）后，结论会变吗？
7. 没有参考基因组（真实场景）时，**无参指标还能选对赢家吗**？
8. 综合质量、成本（$/Gb）、算力（RAM/时间），**给定预算应该选哪种测序+组装策略**？

## 4. 实验矩阵（分层设计，避免全交叉爆炸）

采用**分层设计**而非全交叉——主网格扫覆盖度×工具，难点轴在固定覆盖度下做定点对照。这本身就是规范的实验设计，是"严谨"的体现。

### 4.1 轴定义

| 轴 | 取值 | 备注 |
|---|---|---|
| 测序技术 | Illumina(PE150) / ONT-R9(~90%) / ONT-R10(~99%) / HiFi(~99.9%) | 含技术代次 |
| 覆盖度 | 5× / 10× / 20× / 30× / 50× | 出饱和曲线、找拐点 |
| 组装器 | SPAdes(短) / Flye·Raven·miniasm(长) / hifiasm(HiFi) / Unicycler(hybrid) | 技术×工具解耦 |
| 参考基因组 | 原始 E.coli / 重复注入 v1·v2 / 含质粒版 | 难点轴 |
| 抛光 | 不抛光 / Racon(CPU) / Medaka(GPU)×1 / Medaka(GPU)×2 | GPU 轴 |
| 随机种子 | {13, 17, 19} | 重复实验出置信区间 |

### 4.2 分层网格

- **L1 主网格（原始参考）**：技术 × 覆盖度 × 组装器 —— 出 Pareto、饱和曲线、技术×工具方差归因。约 40–50 组装，头部配置各 3 个种子。
- **L2 抛光子矩阵**：ONT-R9/R10 × 覆盖度 × {无/Racon/Medaka×1/Medaka×2}（GPU）。
- **L3 难点定点（固定 30×）**：重复 v1/v2 + 质粒版，每技术各跑一遍 —— 出重复分辨率、质粒回收。
- **L4 鲁棒性定点（P2，固定 30×）**：脏数据 × 各技术。

> E. coli 仅 4.6 Mb，单组装多为分钟级；全矩阵在一台普通机器 + 你们的 48G GPU 上 2 周内可跑完——"小基因组也能跑满上百组实验"本身强化"低算力"卖点。

## 5. 接口契约（**Day 1 全员 2 小时定死，这是并行的基石**）

所有人按统一约定产出文件，下游靠"文件名 + CSV 列名"自动对接，**谁都不用等谁的代码**。

### 5.1 统一 run-id

```
{tech}_{cov}x_{assembler}_{ref}_s{seed}
例：ont_r10_30x_flye_orig_s13
```

### 5.2 目录与命名（在现有结构上扩展）

```
data/reference/   ecoli.fasta | ecoli_rep_v1.fasta | ecoli_rep_v2.fasta | ecoli_plasmid.fasta
data/reads/       {tech}_{cov}x[_1/_2].fastq   （illumina 双端带 _1/_2）
assemblies/       {run-id}/assembly.fasta  (+ assembly_graph.gfa 若有)
results/tables/   read_stats.csv | assembly_metrics.csv | resource_log.csv |
                  polish_metrics.csv | downstream_metrics.csv | cost_model.csv |
                  final_recommendation.csv
results/figures/  *.png
```

### 5.3 主指标表 schema（`assembly_metrics.csv`，全员汇聚于此）

```
run_id, tech, generation, coverage, assembler, reference, seed,
n50, contigs, largest_contig, total_length, genome_fraction_pct,
mismatches_per_100kbp, indels_per_100kbp, misassemblies,
busco_complete_pct, merqury_qv, ram_mb, wall_sec
```

### 5.4 配置驱动

矩阵写在 `config/matrix.yaml`，Snakemake 按它展开所有任务；随机种子集中在 `config/seeds.yaml`。**改矩阵只改 YAML，不改代码。**

## 6. 四人并行分工（每人 Day 1 就能独立开工）

> **并行保证**：每个人的起步任务都对着**仓库里已存在的某个产物**开发，无需等待上游。
> **资源错峰**：B 吃 CPU、D 吃 GPU，互不抢占。

### 成员 A —— 模拟数据 & 基因组工程

- **负责创新点**：3（重复，数据侧）、7（质粒）、10（脏数据，P2）
- **Day 1 起步依据**：已有 `data/reference/ecoli.fasta`，可立即写模拟脚本，不依赖任何人。
- **任务清单**：
  1. 覆盖度梯度 × 技术代次模拟脚本（ART → Illumina；Badread 调 identity → ONT-R9/R10/HiFi）。
  2. 构建 3 个变体参考：重复注入 v1/v2（不同长度/拷贝数的串联+散在重复）、含质粒版。
  3. （P2）脏数据集：Badread `--chimeras/--junk_reads/--glitches`。
  4. `stat_reads.py` 升级，按 run-id 批量统计 → `read_stats.csv`。
- **交付**：reads 矩阵、4 个参考、`read_stats.csv`、读长分布图。
- **答辩讲**：数据如何生成、重复/质粒怎么构造、为什么这样设计变量。

### 成员 B —— 组装矩阵 & 工具解耦（CPU 重活）

- **负责创新点**：2（技术×工具解耦）、8（组装图可视化）
- **Day 1 起步依据**：用**已有的 30× reads**先把 6 个组装器脚本全部跑通验证，不等 A 的全矩阵。
- **任务清单**：
  1. 封装 SPAdes / Flye / Raven / miniasm / hifiasm / Unicycler 统一调用接口（输入 run-id，输出标准路径）。
  2. 跑 L1 主网格 + Hybrid 配比（Illumina30× + ONT{5/10/20×}）。
  3. 用 `/usr/bin/time -v` 记录 RAM/时间 → `resource_log.csv`。
  4. 导出组装图 GFA，用 Bandage 出"短读长缠成团 vs 长读长一条线"对比图。
- **交付**：assembly 矩阵、`resource_log.csv`、Bandage 图、技术×工具差异原始数据。
- **答辩讲**：不同组装器原理（de Bruijn vs OLC）、为什么短读长会碎（配图）。

### 成员 C —— 评估 & 分析（项目大脑）

- **负责创新点**：1（Pareto）、3（重复分辨率，分析侧）、6（下游效用，分析侧）、7（质粒，测量侧）、9（无参审计，P2）
- **Day 1 起步依据**：用**已有的 assemblies + `assembly_metrics.csv`**把全套评估/分析/画图先跑通，数据到了直接换输入。
- **任务清单**：
  1. 统一评估封装：QUAST + BUSCO(bacteria_odb10) + Merqury(无参 QV)。
  2. 分析：Pareto 前沿、最小够用覆盖度（拐点检测）、技术×工具方差归因、重复分辨率曲线、质粒回收率。
  3. 下游效用：Prokka 基因注释 → 完整基因恢复率（变异检测部分与 D 协作）。
  4. （P2）无参指标审计：盲排名 vs QUAST 真排名一致性。
  5. 全部成图（带置信区间，多种子）。
- **交付**：全部分析表 + 所有结果图 + 统计结论。
- **答辩讲**：指标含义、Pareto 怎么读、"高 N50 ≠ 能用"的证据。

### 成员 D —— 工程化 & GPU/ML（复现 + 加速 + 统稿）

- **负责创新点**：4（harness+成本模型）、5（GPU 神经抛光）、6（Clair3-GPU 变异检测）、11（推荐器，P2）
- **Day 1 起步依据**：用**已有的 `nanopore_30x` 组装 + `nanopore_polished_v2`**先把 Medaka(GPU) 抛光跑通；同时搭 harness 骨架。
- **任务清单**：
  1. **GPU 环境**：Medaka / Clair3 / （可选 DeepConsensus）装好，验证吃满 48G。
  2. L2 抛光矩阵：ONT × 覆盖度 × {无/Racon/Medaka×1/×2}，记录质量提升 + GPU 时间 → `polish_metrics.csv`。
  3. Clair3(GPU) 变异检测，配合 C 的下游效用 → `downstream_metrics.csv`。
  4. Snakemake harness：`config/matrix.yaml` 驱动，固定种子，一键 `snakemake --cores N` 重跑全流程。
  5. 成本模型 `cost_model.csv`（文献 $/Gb）。
  6. （P2）测序策略推荐器：输入预算/基因组大小/精度需求 → 输出推荐策略。
  7. 统稿论文/PPT，统一术语与故事主线。
- **交付**：可复现流水线、抛光/下游/成本表、（P2）推荐器、report/PPT。
- **答辩讲**：GPU 神经抛光能否让 ONT 追平 HiFi、可复现性、最终决策建议。

## 7. 两周并行时间表（泳道 + 里程碑）

```
        第 1 周：各自对着已有产物搭好并验证脚本（完全并行）
A  ──[模拟脚本 + 4 个参考构建]───────────────▶ M1: reads 矩阵就绪
B  ──[6 组装器封装 + Bandage 图跑通]─────────▶ M2: 组装脚本就绪
C  ──[QUAST/BUSCO/Merqury + 全分析+画图跑通]─▶ M3: 评估/分析就绪
D  ──[Medaka-GPU 抛光跑通 + harness 骨架]────▶ M4: GPU+harness 就绪

        第 2 周：跑满矩阵 → 出结论 → 写
A→B  ── 接真 reads 跑满 L1/L2/L3 主矩阵（B 主跑，D 跑抛光/GPU）
   →C  ── C 评估全矩阵 + 下游效用 + Pareto/方差/重复/质粒 定论
        ── (P2) ③无参审计 / ④脏数据 / ⑥推荐器（有时间才做）
全员 ── 各写自己章节 → D 统稿 + 出 PPT + 答辩排练
```

里程碑硬节点：
- **第 1 周末（M1–M4）**：四条线脚本全部跑通（对样板数据），开始接真数据。
- **第 2 周中**：L1+L2+L3 主矩阵跑完，核心图表出齐。
- **第 2 周末**：P2 收尾或砍掉、论文/PPT 定稿。

## 9. 风险与降级预案

| 风险 | 预案 |
|---|---|
| 进度紧 | **按 P2→P1 顺序砍**：先砍 ⑥推荐器、④脏数据、③无参审计；P0 必保。 |
| 某工具装不上 | 该技术保留 ≥1 个可用组装器即可（如 ONT 只剩 Flye）；用 conda/bioconda 优先。 |
| GPU 工具(Clair3/Medaka)环境难搞 | 抛光保 Medaka 一项即可；Clair3 变异检测可降级为"仅基因恢复"。 |
| 矩阵太大跑不完 | 多种子只对头部配置；难点轴只固定 30×；覆盖度可砍到 {10/30/50}。 |
| 结果不符预期 | 照实解释——梯度/机制本身就是结论，不追求"完美" |

## 10. 答辩故事主线（一句话串起来）

> 我们没有停在"HiFi 最好"，而是构建了一个**低算力、可复现、控制变量清晰的多代测序组装基准**：通过覆盖度梯度与重复/质粒难点**重新制造区分度**，用因子设计**分离"测序技术"与"组装工具"的贡献**，用 GPU 神经抛光检验**廉价 ONT 能否追平昂贵 HiFi**，再把"组装质量"升维到**基因恢复与变异检测等下游实用价值**，最终综合质量、成本与算力给出**预算驱动的测序策略建议**。
