# 修复执行计划（你照着跑）

> 配套：`docs/执行手册-逐步命令.md`（完整流程）。本文档**只解决上次检查发现的故障**，按"杠杆"从高到低排。
> 上次检查结论：HiFi 那条线真跑通了（5/10/20/30/50× 干净饱和曲线，20× 拼成单条），但**核心"二代 vs 三代"对比为空**，卡在三件事：①沙箱杀长读长组装 ②ONT 数据没齐 ③GPU 环境没建。

## 诊断回顾（证据）

| 故障 | 证据 | 影响 |
|---|---|---|
| ①**沙箱拒绝 socket** | `PermissionError: [Errno 1] Operation not permitted` @ `SocketListener.setsockopt` | Flye consensus 阶段崩 → 所有 ONT/长读长组装挂掉 |
| ②**raven/miniasm 没装** | `.conda/bench/bin/{raven,miniasm}` 不存在 | ONT 的 3 个组装器里 2 个直接报"工具找不到" |
| ③**ONT 数据缺** | `ont_r10` 0 文件；`ont_r9_30x.fastq.partial`；无 50× | 三代代次轴 + ONT 组装全空 |
| ④**GPU 环境没建** | `.conda/gpu` 不存在 | Medaka 抛光(创新5)、Clair3(创新6)做不了 |
| ⑤**BUSCO/Merqury 没跑** | 主表 `busco_complete_pct` 整列空 | 无参评估缺失 |

> 注：`minimap2` 现在**已装好**（2.31），所以 ① 的真正总闸是 **socket 沙箱**，不是 minimap2。

约定：`✅检查点` 过了才往下走。命令都从项目根目录、用环境内解释器运行。

## F2 — 解除沙箱（总闸，~5 min）

故障 ① 是运行环境**禁止建 socket**，Flye 的多进程 consensus 因此崩溃（hifiasm 不用 socket 才侥幸跑通）。

- [ ] **F2.1 自检：当前 shell 能不能建 socket**

```bash
.conda/bench/bin/python -c "import multiprocessing.connection as c; l=c.Listener(); print('SOCKET OK')"
```
- 打印 `SOCKET OK` → 你的终端没问题，直接进 F3。
- 报 `PermissionError: [Errno 1] Operation not permitted` → **你在受限/沙箱环境里跑**。

- [ ] **F2.2 若上面报错：换到普通终端**

不要通过任何"沙箱化/受限"的封装来跑组装。直接在你自己的系统终端里：
```bash
cd /home/zj/cw/multi-gen-sequencing-benchmark
source ~/.bashrc   # 或确保 conda 可用
.conda/bench/bin/python -c "import multiprocessing.connection as c; c.Listener(); print('SOCKET OK')"
```
（若你是在 Claude Code 里跑，这一步及之后的组装请用 `!` 前缀在本机终端执行，或关闭沙箱。）

- [ ] **F2.3 Flye 冒烟：确认长读长组装真能跑通**

```bash

  --tech hifi --coverage 30 --assembler flye --reference orig --seed 13 --threads 8
ls -l assemblies/hifi_30x_flye_orig_s13/assembly.fasta
```

✅**检查点 F2：** Flye 跑完、产出非空 `assembly.fasta`、**不再报 socket 错**。这是后面一切长读长组装的前提。

## F4 — 跑 Illumina + ONT 组装矩阵（核心对比，~3–5 h，需 F2 已解沙箱）

> HiFi 已跑过会自动 skip；hybrid 单独在 F4.2 处理。**务必在能建 socket 的终端里跑。**

- [ ] **F4.1 主循环：illumina(spades) + ont_r9/ont_r10(flye/raven/miniasm)**

```bash
.conda/bench/bin/python - <<'PY' 2>&1 | tee logs/assemble_fix.log
import yaml
from pathlib import Path
from scripts.benchmark_common import expand_matrix, make_run_id
from scripts.run_assemblers import run_one
cfg = yaml.safe_load(open("config/matrix.yaml"))
seed = yaml.safe_load(open("config/seeds.yaml"))["default_seed"]
want = {"illumina", "ont_r9", "ont_r10"}     # hifi 已完成；hybrid 见 F4.2
for r in expand_matrix(cfg, seed):
    if r["reference"] != "orig" or r["tech"] not in want:
        continue
    rid = make_run_id(r["tech"], r["coverage"], r["assembler"], r["reference"], r["seed"])
    if Path("assemblies", rid, "assembly.fasta").is_file():
        print("SKIP", rid); continue
    try:
        run_one(r["tech"], r["coverage"], r["assembler"], r["reference"], r["seed"], threads=8)
        print("OK", rid)
    except Exception as e:
        print("FAIL", rid, repr(e))
PY
grep -c OK logs/assemble_fix.log; echo "FAILs:"; grep FAIL logs/assemble_fix.log
```

## F6 — 重建主表（含 BUSCO/Merqury，~2–3 h，需联网下 BUSCO 谱系）

> BUSCO 首次要联网下载 `bacteria_odb10`。**这次别加 `--skip-busco`。**

```bash
# 备份旧表，从零重建
mv results/tables/assembly_metrics.csv results/tables/assembly_metrics_partial.bak 2>/dev/null || true
.conda/bench/bin/python - <<'PY' 2>&1 | tee logs/evaluate_fix.log
import subprocess
from pathlib import Path
from scripts.benchmark_common import parse_run_id
REF = {"orig":"ecoli","rep_v1":"ecoli_rep_v1","rep_v2":"ecoli_rep_v2","plasmid":"ecoli_plasmid"}
for d in sorted(Path("assemblies").iterdir()):
    asm = d / "assembly.fasta"
    if not asm.is_file():
        continue
    try:
        ctx = parse_run_id(d.name)
    except ValueError:
        continue  # 跳过旧的非 run-id 目录
    subprocess.run([".conda/bench/bin/python","scripts/evaluate_assembly.py",
        "--run-id",d.name,"--assembly",str(asm),
        "--reference",f"data/reference/{REF[ctx['reference']]}.fasta",
        "--out-dir",f"quast_reports/{d.name}","--threads","8"], check=False)
PY
echo "主表行数：$(($(wc -l < results/tables/assembly_metrics.csv)-1))"
echo "技术种类："; awk -F, 'NR>1{print $3}' results/tables/assembly_metrics.csv | sort -u
echo "BUSCO 非空行：$(awk -F, 'NR>1 && $16!=""' results/tables/assembly_metrics.csv | wc -l)"
```
✅**检查点 F6：** 主表的 `tech` 列出现 **illumina / ont_r9 / ont_r10 / hifi（+hybrid）多种**；`busco_complete_pct` 大量有值。
